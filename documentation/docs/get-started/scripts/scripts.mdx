# Available Scripts

## `npm i`

Install node modules, pod files , update pod repo and check androidX jetifier.

## `npm start`

Run the React Native packager in dev mode. It is required to run the app using
any of the commands below.

Sometimes you may need to reset or clear the React Native packager's cache. To
do so you can pass the `--reset-cache` flag to the start script:

```sh
npm start -- --reset-cache
```

## `Android`

### `npm run start:android:brand1:prod`

### `npm run start:android:brand1:dev`

### `npm run start:android:brand1:live`

Build and run the app on a connected Android device or emulator. Requires an
installation of Android build tools (see
[React Native docs](https://reactnative.dev/docs/getting-started) for detailed
setup). It uses the relevant environment config file from the `env` directory
(see [Config](config.md)).

This command supports a number of different arguments:

```sh
# run a specific variant of the app
npm run start:android:brand1:prod -- -- --mode='brand1Release'
```

### `npm run build:android:brand1:prod`

### `npm run build:android:brand1:dev`

### `npm run build:android:brand1:live`

Build an Android `.apk` file and output it to
`./android/app/build/outputs/apk/brand1/release/app-brand1-release.apk`. It uses
the relevant environment config file from the `env` directory (see
[Config](config.md)).

### `npm run bundle:android:brand1:live`

Build an Android `.aab` file and output it to
`./android/app/build/outputs/bundle/brand1Live/app-brand1-live.aab`. It uses the
relevant environment config file from the `env` directory (see
[Config](config.md)).

## `IOS`

### `pod update hermes-engine --no-repo-update`

Install pod install.

## `Other Comments`

### `npm test`

Run the [jest](https://github.com/facebook/jest) test runner.

### `npm check-types`

Run the [typescript](https://www.typescriptlang.org/) type checker.

### `npm run lint`

Run the [eslint](https://github.com/eslint/eslint) linter.

### `npm run format`

Run the [prettier](https://prettier.io/) Prettier.

### `npm run install:npm`

Install the correct npm version using the `engines` field from `package.json`.

### `npm run release`

Start an app release from a release branch (see [Contributing](contributing.md)
guidelines).

### `npm audit fix --force`

Fix vulnerabilities.
